## QUALITY ASSURANCE PROCESS

### Automated Quality Checks

Every output is automatically checked for:

1. **Clarity Check** (0-10 score)
   - Sentence length <25 words average
   - No jargon or complex terminology
   - Active voice preferred
   - Clear structure and flow

2. **Compliance Check** (0-10 score)
   - All regulatory requirements met
   - Certifications current
   - Legal terms included
   - Audit trail present

3. **Professionalism Check** (0-10 score)
   - No spelling errors
   - Consistent formatting
   - Professional tone
   - Proper spacing and alignment

4. **Completeness Check** (0-10 score)
   - All required fields present
   - No placeholder text
   - All sections included
   - Supporting documents attached

5. **Accuracy Check** (0-10 score)
   - Data is correct
   - Calculations verified
   - Totals match
   - No inconsistencies

### Scoring System

- **9-10**: Exceeds expectations - ready to publish immediately
- **8-9**: Meets requirements - ready to publish with minor review
- **7-8**: Acceptable with improvements - address feedback before publishing
- **5-7**: Below standard - significant improvements needed
- **<5**: Unacceptable - major revisions required

### Pass/Fail Criteria

**PASS** (Ready to Publish):
- Overall score ≥8.0
- All individual checks ≥7.0
- No critical issues
- All required fields present
- Data is accurate

**FAIL** (Needs Improvement):
- Overall score <8.0
- Any check <7.0
- Critical issues present
- Missing required fields
- Data inaccuracies

## IMPROVEMENT LOOP

### When Quality Check Fails

1. **Identify Issues**
   - Review quality check results
   - Understand what failed
   - Get specific feedback

2. **Make Improvements**
   - Fix clarity issues
   - Add missing compliance
   - Improve professionalism
   - Complete missing fields
   - Correct inaccuracies

3. **Re-Check Quality**
   - Run quality check again
   - Verify all issues fixed
   - Confirm score ≥8.0

4. **Publish**
   - Once quality check passes
   - Output is ready for use
   - Track in audit trail

### Continuous Improvement

- Track which issues are most common
- Update templates to prevent issues
- Train team on quality standards
- Monitor user feedback
- Refine standards based on real-world feedback

## API PROCEDURES

### Check Output Quality

```bash
POST /api/trpc/qa.checkQuality
{
  "outputType": "quote",
  "content": "...",
  "metadata": {
    "total": 5000,
    "client": "John Smith"
  }
}
```

**Response**:
```json
{
  "checkId": "qa-1234567890",
  "outputType": "quote",
```
